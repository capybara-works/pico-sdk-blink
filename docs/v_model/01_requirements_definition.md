# 要件定義書 (Requirements Definition Document)

## 1. はじめに
本ドキュメントは、Raspberry Pi Pico (RP2040) を使用した「pico-sdk-blink」プロジェクトの要件を定義するものである。

## 2. プロジェクト概要
### 2.1 背景・目的
*   組み込みシステム開発の学習用として、最小構成のプロジェクトを提供する。
*   ハードウェア（Wokwiシミュレータ）とソフトウェア（Pico SDK）の連携を理解する。
*   CI/CDパイプラインによる自動化された品質保証プロセスを確立する。

### 2.2 対象システム
*   Raspberry Pi Pico (RP2040) 搭載ボードを使用したLED制御システム。

## 3. 機能要件
### 3.1 LED制御機能
*   **REQ-F-001**: システムは、指定されたGPIOピンに接続されたLEDを周期的に点滅（点灯・消灯）させなければならない。
*   **REQ-F-002**: 点滅の周期は、点灯250ms、消灯250ms（合計500ms）でなければならない。

### 3.2 ログ出力機能
*   **REQ-F-003**: システムは、LEDの状態変化に合わせて、シリアル通信経由でログを出力しなければならない。
*   **REQ-F-004**: LED点灯時には "LED on"、消灯時には "LED off" という文字列を出力すること。

## 4. 非機能要件
### 4.1 ハードウェア要件
*   **REQ-NF-001**: マイクロコントローラとして Raspberry Pi Pico (RP2040) を使用すること。
*   **REQ-NF-002**: LEDはオンボードLED (GP25) を使用すること。

### 4.2 ソフトウェア要件
*   **REQ-NF-003**: 開発言語は C++ (C++17 Standard) を使用すること。
*   **REQ-NF-004**: ビルドシステムとして CMake を使用すること。
*   **REQ-NF-005**: Raspberry Pi Pico SDK を使用すること。

### 4.3 開発・運用環境要件
*   **REQ-NF-006**: Wokwiシミュレータ上で動作確認が可能であること。
*   **REQ-NF-007**: GitHub Actions を使用したCI/CDパイプラインにより、ビルドとテストが自動化されていること。
